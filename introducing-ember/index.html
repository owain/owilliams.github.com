Classes & Objects

Classes Extend

App.Person = Ember.Object.extend({
  firstName: null,
  lastName: null
});


var bieber = App.Person.create({
  firstName: "Justin",
  lastName: "Bieber"
});
Provides simple classes & inheritance

Mixins

App.Canadian = Ember.Mixin.extend({
  says: function(something) {
    alert(this.get("firstName") + " says: " + something + ", Eh!");
  }
});

App.Person = Ember.Object.extend(App.Canadian, {
  firstName: null,
  lastName: null
});

var gretzky = App.Person.create({
  firstName: "Wayne",
  lastName: "Gretzky"
});

gretzky.say("I play hockey"); // alerts "Wayne says: I play hockey, Eh!"
Flexibility & code reuse of Mixins

Objects - get/set

App.Person = Ember.Object.extend({
  firstName: null,
  lastName: null,
});

var john = App.Person.create({
  firstName: "John",
  lastName: null
});

john.get("lastName"); // => null

john.set("lastName", "Candy");

john.get("lastName"); // => "Candy"
Using get/set allows Ember to be very intelligent with performance & updating

Computed Properties

A property that derives its value by executing a function

Computed Property Example

App.Person = Ember.Object.extend({
  firstName: null,
  lastName: null,

  fullName: function() {
    return this.get("firstName") + ' ' + this.get("lastName");
  }.property("firstName", "lastName")
});

var avril = App.Person.create({
  firstName: "Avril",
  lastName: "Lavigne"
});

avril.get("fullName") // "Avril Lavigne"
  • only re-evaluates when the dependant properties change
  • propegate through system if dependant on CP
    • Run-loop - updates once
    • Lazy computation
    • Cached

Views & Templates

Templates

  • Use Handlebars.js to define your templates
  • Describes HTML output
  • Bindings aware

Templates Example

< div class="person-name" >
	Hello, {{firstName}} {{lastName}}!
< /div >
Expressions take data from controllers, place them into HTML - automatically keeping them updated.

View

  • Handles user interaction (clicks, events)
  • Adapts styling according to data
  • Useful for reusable components
  • Usually small & sometimes not needed
Translates primitive events (like clicks, taps, and swipes) into semantic events

View Example

App.MooseView = Ember.View.extend({
  templateName: 'moose',

  classNameBindings: ['isAngry'],
  isAngry: true,

  click: function(evt) {
    controller.hug(); // => "You hugged a moose!"
  }
});

Models & Data

A model is an object that stores persistent state

In essence a model can be just an Ember Object that takes advantage for Computed Properties, Binding, etc

Ember Data

An official data persistence library

Goal of to make API communication & persistent state management easier

  • Handles you Ajax, model state, etc
  • Won't cover in great detail
  • More in a bit

Controllers

Controllers in Ember

  • Deliver data to views & templates
  • Decorates model data with display logic
  • Can handle actions
  • Long lived

Ember.Controller

Proxies to its own properties

Ember.ObjectController

Proxies to its itself, and then to its model's properties

Ember.ArrayController

Proxies to its itself, and then to its models properties, which is an Array of models

Quick Recap

  • Model holds your data
  • Controller handles the communication with the Model
  • Controller can also decorate the data with display data
  • Templates talk to the Controller to get the data

Router

Router

The Router maps the browser's URL to Application State

  • Runs code when URLs are visited
  • Router is the glue to put your application together
  • Loads data and assigns it to a controller
  • controllers don't load their own data usually
App.Router.map(function() {
  this.route("about", { path: "/about" });
  this.route("favorites", { path: "/favs" });
});
Familiar to Rails Developers 
App.AboutRoute = Ember.Route.extend({
  setupController: function(controller) {
    // Set the IndexController's `title`
    controller.set("title", "My App");
  }
});
